            # About Slice Stores

            Slice stores are optional [Zustand](https://zustand.docs.pmnd.rs/learn/getting-started/introduction) helpers for shared UI state that make implementing interactions easy (for example toast messages). They are designed to be simple, global, and easy to consume from both React and non-React code.

            ## Optional dependency
            Slice components and hooks do not require stores. Install Zustand only if you want to use store paths.
        </Unstyled>
        <RenderSource language='bash'>
            $ npm install zustand
        </RenderSource>
        <Unstyled>
            ## Singleton pattern
            Slice stores are created once at module load and reused across all imports in your app runtime.  
            That means any part of your app importing the same store reads/writes the same shared state instance.

            ## React (reactive) vs imperative (non-reactive)
            - Reactive store hooks subscribe components to selected state and trigger re-renders when selected values change.
            - Imperative helpers read/write state directly and are meant for utilities, services, and other non-React paths.
        </Unstyled>
        <RenderSource language='tsx'>
            {`
import { useToast, useToastActions } from '@apple-pie/slice/stores/toast';

function SaveButton() {
  const toast = useToast();
  const actions = useToastActions();

  return (
    <button
      type="button"
      onClick={() => actions.push({ message: 'Saved', type: 'success' })}
    >
      {toast?.message ?? 'Save'}
    </button>
  );
}
`}
        </RenderSource>
        <RenderSource language='tsx'>
            {`
import { toastActions, getToast } from '@apple-pie/slice/stores/toast';

export function notifyError() {
  toastActions.push({ message: 'Something failed', type: 'error' });
}

export function currentToastMessage() {
  return getToast()?.message ?? null;
}
`}
        </RenderSource>
