        # Camera
        Renders a live camera preview with optional microphone access, inline capture controls, a poster fallback, and a picture-in-picture snapshot preview. This component is a special case in the library because it has both a prop interface and an imperative ref interface.

        ## Base element
        `HTMLDivElement`

        The internal `HTMLVideoElement` is exposed through the forwarded ref as `CameraElement.video`.
        
        ## Base Interfaces
        `CameraProps` and `CameraElement` (forwarded ref)

        ## Why a ref API exists
        Camera streams are inherently imperative browser objects. The component emits a few callbacks such as `onVideoStream` and `onSnapshot`, but it does not try to mirror every stream detail into React state or emit high-frequency state events. Instead, the forwarded ref is the primary way to:

        - start or stop the stream
        - toggle video or microphone tracks
        - inspect the underlying `MediaStream`
        - access raw `MediaStreamTrack` data
        - capture snapshots on demand

        ## Using The Ref
        Because stream details are not mirrored into React events, parent components should treat the camera ref as the source of truth for low-level media state.

{`
import { useEffect, useRef, useState } from 'react';
import { Camera } from '@apple-pie/slice/components/Camera';
import type { CameraElement } from '@apple-pie/slice/components/Camera';

export function CameraRefExample() {
  const cameraRef = useRef<CameraElement | null>(null);
  const [streamId, setStreamId] = useState<string>('none');

  const refreshInfo = () => {
    setStreamId(cameraRef.current?.stream?.id ?? 'none');
  };

  useEffect(() => {
    refreshInfo();
  }, []);

  return (
    <>
      <Camera
        ref={cameraRef}
        width={400}
        height={320}
        sessionSettings={{
          videoDeviceId: 'preferred-video-device-id',
          micDeviceId: 'preferred-mic-device-id',
        }}
      />
      <button
        onClick={async () => {
          await cameraRef.current?.toggleVideo?.();
          refreshInfo();
        }}
      >
        Toggle Video
      </button>
      <button
        onClick={() => {
          cameraRef.current?.toggleMic?.();
          refreshInfo();
        }}
      >
        Toggle Mic
      </button>
      <button
        onClick={() => {
          cameraRef.current?.snapshot?.();
          refreshInfo();
        }}
      >
        Take Snapshot
      </button>
      <div>Current stream: {streamId}</div>
    </>
  );
}
`}

        ## Changing Selected Devices
        Device selection is controlled by the application, not `Camera`. `onChangeSettings` does not update device selection - it only a signals that the user triggered the settings action.

        The expected flow is:
            
        #### 1. User clicks "settings" action button
        This fires `onChangeSettings`, providing:
        - the current `sessionSettings`
        - the current cached device list as `MediaDeviceInfo[] | Error`

        #### 2. App displays settings modal / UI
        The app is responsible for using the information provided to display and update user selected devices.
            
        #### 3. App updated its internal state and settings prop
        the app then updates it's internal state as needed and passes the selections to `Camera` via the settings prop.
            
        ## Notes
        - The component requests preferred devices from `sessionSettings` up front using `getUserMedia` constraints.
        - If a requested device ID is stale or unavailable, the component retries with default browser devices.
        - Turning video off disables the video track instead of stopping the full stream, so audio can remain active.
        - Audio capture and local audio playback are separate concerns. Use <code>startAudioMuted</code> to disable the microphone track itself, and <code>playLocalAudio</code> to control whether the live preview outputs audio to local speakers.
        - `stopCamera()` is the method that tears down the full stream.
