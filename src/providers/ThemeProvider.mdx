        # ThemeProvider
        Provides Slice theme behavior by writing `data-slice-theme` on `document.documentElement`, rendering a scoped wrapper with `data-slice-theme-scope`, and loading Slice CSS token/type styles.
        When the provider resolves an explicit, initial, or system theme on the client, it also persists a `slice-theme` cookie so SSR helpers can read the same resolved theme on the server for the next request.
        It also seeds Slice's theme hooks with the server-resolved startup theme and startup system flag through internal context, which keeps `useTheme()` and `useObserveTheme()` aligned during hydration.
        That internal context also tracks whether the active theme is still system-derived, and `useTheme().set(...)` / `useTheme().toggle()` update that flag when switching between `system` and explicitly named themes.

        ## Base element
        `div`

        ## Interface
        `ThemeProviderProps`

        <Unstyled>
        ## SSR Notes
        - `ThemeProvider` is the client runtime piece.
        - Pass the server-resolved `initialTheme` and `initialSystem` into the provider when using SSR.
        - If `initialSystem` is omitted, Slice treats the startup theme as system-derived by default.
        - `useTheme()` and `useObserveTheme()` read that startup value from the provider's internal context before switching over to DOM observation.
        - The provider context also keeps a live `systemTheme` flag so consumers can tell whether the current theme came from `prefers-color-scheme` or from an explicit client-side selection.
        </Unstyled>
