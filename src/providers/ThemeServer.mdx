        # ThemeServer
        Server-safe helpers for SSR-enabled environments such as Next.js. These helpers resolve a startup theme before hydration and provide the root HTML attributes needed for first-paint theming.
        They are designed to pair with the `slice-theme` cookie written by `ThemeProvider` on the client.

        ## Import Paths
        Use the server helpers from:

        `@apple-pie/slice/providers/themeServer`

        Use the client runtime provider from:

        `@apple-pie/slice/providers/ThemeProvider`

        ## Recommended SSR flow
        - Read the `slice-theme` cookie on the server.
        - Pass that value into `resolveInitialTheme()`.
        - Spread `getThemeHtmlAttributes(initialTheme)` onto the root `html` element.
        - Pass the same `initialTheme` into `ThemeProvider` on the client.
        - Let `ThemeProvider` refresh the cookie after hydration so future requests keep the same theme.

        ## Cookie behavior
        - Cookie name: `slice-theme`
        - Cookie value: the resolved Slice theme name such as `lightMode` or `darkMode`
        - Cookie attributes: `path=/`, `max-age=31536000`, `SameSite=Lax`
        - Intended use: lets SSR code read the last client-selected theme and render the correct root theme attribute before hydration


        ## Next.js Example
        </Unstyled>

        <RenderSource language={"tsx"}>
        ```
        import { cookies } from 'next/headers';
        import {
            getThemeHtmlAttributes,
            resolveInitialTheme,
        } from '@apple-pie/slice/providers/themeServer';
        import { ThemeProvider } from '@apple-pie/slice/providers/ThemeProvider';
        
        export default async function RootLayout({
            children,
        }: {
            children: React.ReactNode;
        }) {
            const cookieStore = await cookies();
        
            const initialTheme = resolveInitialTheme({
                cookieTheme: cookieStore.get('slice-theme')?.value,
                fallbackTheme: 'lightMode',
            });
        
            return (
                <html lang="en" {...getThemeHtmlAttributes(initialTheme)}>
                    <body>
                        <ThemeProvider initialTheme={initialTheme} global>
                            {children}
                        </ThemeProvider>
                    </body>
                </html>
            );
        }
        ```
        </RenderSource>

        <Unstyled>
        ## Notes
        - `ThemeProvider` remains the client-side runtime piece.
        - The server helpers exist to make the first HTML response theme-aware before React hydration begins.
        - The expected cookie name is `slice-theme`.
        - If you only set the theme in a client effect, the first paint may occur before the theme attribute is applied and you get a UI flicker.
