        # LocalDBStore
        Slice's `localDBStore` is a global Zustand store that manages one or more `IndexedDB` connections and keeps a reactive in-memory snapshot of each store's records. IndexedDB stores have no size limit (other than available disk space) and can store complex objects like files, etc. that local store cannot.
            
        The goal is to simplify using the browser's local indexed database as state for your react apps. If you need full control over indexedDB, you can always use the `IndexedDB` class directly. 
            
        ## Adding an indexedDB store
        To create or access an IndexedDB local store, call `addStore(name, options)`. If it does not exist, a new local store will be created. Otherwise, it will be connected and state will be refreshed.
            
        When you add a store, there are a couple of important options to keep in mind:
        - `key?: string` - this key will be used to derive record keys from object values.
        - `create?: boolean` - use this flag to prevent store creation when the database or object doesn't exist.

        <RenderSource language={'tsx'}>
        ```
        import { useEffect, type ReactNode } from 'react';
        import { useManageLocalDB } from '@apple-pie/slice/stores';
            
        export type User = {
            id: string; // "id" will be used as the key for the DB adds, updates, etc.
            name: string;
            email: string;
        };
        
        export function App({ children }: { children: ReactNode }) {
            const { addStore } = useManageLocalDB();
                    
            useEffect(() => {
                void addStore('users', { key: 'id' });
            }, []);
        
            return children;
        }
        ```
        </RenderSource>

        ## Adding and removing values
        
        Once you have added the store, you can add, update, and remove values from anywhere in your app by calling the bound store actions.

        <RenderSource language={'tsx'}>
            ```
            import { useLocalDB, useLocalDBValues } from '@apple-pie/slice/stores';

            type User = {
                id: string;
                name: string;
                email: string;
            };

            export function AddUsersExample() {
                const users = useLocalDBValues<User>('users');
                const { add, set, remove } = useLocalDB('users');

                // add user(s) - if a key already exists, add will fail and generate an error
                async function seedUsers() {
                    const result = await add([
                        { id: 'user-1', name: 'doe', email: 'doe@example.com' },
                        { id: 'user-2', name: 'john', email: 'john@example.com' },
                    ]);

                    if (!result.ok) {
                        console.error(result.error.message);
                    }
                }

                // update user(s) - set uses IndexedDB put semantics, so it upserts
                async function updateUser() {
                    const result = await set({
                        id: 'user-1',
                        name: 'doe',
                        email: 'updated@example.com',
                    });

                    if (!result.ok) {
                        console.error(result.error.message);
                    }
                }

                // remove user(s) - remove derives the key from the object value
                async function deleteUser() {
                    const result = await remove({
                        id: 'user-1',
                        name: 'doe',
                        email: 'updated@example.com',
                    });

                    if (!result.ok) {
                        console.error(result.error.message);
                    }
                }

                return (
                    <div>
                        <button onClick={seedUsers}>Seed users</button>
                        <button onClick={updateUser}>Update user-1</button>
                        <button onClick={deleteUser}>Delete user-1</button>
                        <div>Total users: {users.length}</div>
                    </div>
                );
            }
        ```
        </RenderSource>

        ## Accessing indexedDB store values
    
        Store values update every time an action is taken on the IndexedDB store. You can use `useLocalDBValues` anywhere in your app to read the latest cached values.

        <RenderSource language={'tsx'}>
            ```
            import { useLocalDBValues } from '@apple-pie/slice/stores';

            type User = {
                id: string;
                name: string;
                email: string;
            };
    
            export function ListUsers() {
                const allUsers = useLocalDBValues<User>('users'); // the first parameter is always the store name
                const allUsersNamedJohn = useLocalDBValues<User>('users', 'name', 'john');
    
                return (
                    <>
                        All users:
                        { allUsers.map((user: User) => {
                            return <div key={user.id}>{user.name}</div>
                        })}
                        Users named John
                        { allUsersNamedJohn.map((user: User) => {
                            return <div key={user.id}>{user.name}</div>
                        })}
                    </>
                );
            }
        ```
        </RenderSource>

        ## Runtime rules and edge cases
        - `useLocalDBValues(...)` reads from the cached Zustand snapshot, not directly from IndexedDB on every render.
        - `useLocalDBValues(name)` always returns an array.
        - `useLocalDBValues(name, key)` compares keys with IndexedDB-aware equality logic, including arrays and `Date` keys.
        - `useLocalDBValues(name, field, value)` filters cached values with strict equality.
        - `setValue` and `removeValue` require `options.key` to be configured when the store is created.
        - `setValue` and `removeValue` expect object values, because the key is derived from a field on each object.
        - `setValue` uses IndexedDB `put`, so it can create or overwrite records.
        - `addValue` uses IndexedDB `add`, so it is insert-only and rejects on duplicate keys.
        - `addValue` does not derive a manual key in `localDBStore`; it relies on the underlying object store configuration such as key path or `autoIncrement`.
        - After successful writes, deletes, or refreshes, the store re-reads all keys and values and replaces the cached `records`.
        - Every Zustand store action returns a result object instead of throwing: either `{ ok: true }`, `{ ok: true, value }`, or `{ ok: false, error }`.
        - All store-managed failures are written to the store `error` field as well as returned in the action result.
        - `removeStore` only unregisters the Zustand entry. It does not delete the IndexedDB database.
        - `destroyStore` deletes the whole IndexedDB database for that store name.
        - The Storybook demo uses a dedicated database named `storybook-localdb-messages-v2` so older demo schemas do not interfere with current behavior.
