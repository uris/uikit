        # WSStore
        Global store for handling websocket events from one or more connections, built with Zustand. The store manages named `WSConnection` instances, emits and keeps a snapshot of the latest unified message, and exposes both reactive hooks and imperative helpers to read messages and manage connections.
        
        The Slice websocket store has a narrow goal: provide a singleton global state for the last unified websocket event received and emit that as state. Storing, persisting, accumulating, and manipulating websocket message history is left to the consuming app.
            
        ## Adding a new websocket connection
        `addConnection` creates a new `WSConnection` and saves it to state. If a connection with the same name already exists, the store closes the old connection before replacing it. The store always creates connections in unified mode, so the options you pass here are the `WSConnection` options except for `unifiedMessages`, which the store manages internally.
        A typical example might look like this:

            <RenderSource language={'tsx'}>
```
import { useEffect, useRef } from 'react';
import {
	useWS,
	useIsConnected,
	useMessage,
	WSStoreConnection,
} from '@apple-pie/slice/stores/WS';

type ChatMessage = { id: string; text: string };

function AppSocket() {
	const connectionRef = useRef<WSStoreConnection | null>(null);
	const { addConnection, removeConnection } = useWS();
	const connected = useIsConnected('chat');
	const anyMessage = useMessage();
	const lastPayload = useMessage<ChatMessage>('message', 'chat');

	useEffect(() => {
		if (connectionRef.current) return;
		connectionRef.current = addConnection('chat', {
			url: 'wss://example.com/chat',
			autoReconnect: true,
			reconnectInterval: 1000,
			reconnectAttempts: 5,
			reconnectFalloff: true,
		});
		return () => {
			removeConnection('chat');
			connectionRef.current = null;
		};
	}, [addConnection, removeConnection]);

	return (
		<div>
			connected: {connected ? 'yes' : 'no'} - message:{' '}
			{typeof lastPayload === 'object' && lastPayload
				? lastPayload.text
				: 'waiting...'}{' '}
			- last type: {anyMessage?.type ?? 'waiting...'}
		</div>
	);
}
```
</RenderSource>
            
            Note: Slice stores are singletons, which means you can safely import `useMessage` anywhere in your app and subscribe to the latest matching websocket update.
            If you plan to make events available globally to your app, make sure to place your connection cleanup where it makes sense.

        ## Removing a connection
        Removing a connection always closes it first, then removes it from the store.

        <RenderSource language={'tsx'}>
{`
import { useWSActions } from '@apple-pie/slice/stores/WS';

useWSActions.removeConnection('chat');
`}
</RenderSource>

        ## Imperative usage outside React
        <RenderSource language={'tsx'}>
{`
import { useWSActions, useLastWSMessage } from '@apple-pie/slice/stores/WS';

useWSActions.addConnection('system', {
	url: 'wss://example.com/system',
	autoReconnect: true,
});

export function getLatestWSType() {
	return useLastWSMessage()?.type ?? null;
}
`}
