        # WebRTCStore
        Global store for managing one or more named `WebRTCConnection` instances with Zustand. The store keeps a simple registry of connections and exposes explicit lifecycle actions for creating, initializing, updating, and removing each connection.

        The store intentionally stays narrow. It does not subscribe to the microphone store or volume store internally. Instead, callers pass the current mic stream and volume level into the store actions when they want to update a connection.

        If you want the microphone store and volume store to be wired automatically, use `useBindWebRTC` instead. The hook is the convenience layer; this store is the explicit lifecycle layer.

        ## Lifecycle
        Typical usage follows this order:

        1. `addConnection(name, options)`
        2. `initializeConnection(name, offerOptions?, bearerToken?)`
        3. `setMicStream(name, stream)` when the processed mic stream changes
        4. `setVolume(name, volume)` when the app volume changes
        5. `removeConnection(name)` during cleanup

        If you prefer not to manage those steps yourself, `useBindWebRTC` wraps this same flow and keeps mic stream and volume synchronized from the stores automatically after binding.

        <RenderSource language={'tsx'}>
        {`
import { useEffect } from 'react';
import {
    useProcessedMicStream,
} from '@apple-pie/slice/stores/microphone';
import {
    useVolume,
} from '@apple-pie/slice/stores/volume';
import {
    useWebRTC,
    useWebRTCConnected,
} from '@apple-pie/slice/stores/WebRTC';

function VoiceCall() {
    const rtc = useWebRTC();
    const processedMicStream = useProcessedMicStream();
    const volume = useVolume();
    const connected = useWebRTCConnected('call');

    useEffect(() => {
        const stream = processedMicStream.current;
        if (!stream) return;

        rtc.addConnection('call', {
            connectionUrl: '/api/rtc',
            micStream: stream,
            volume,
            dataChannels: ['events'],
        });

        void rtc.initializeConnection('call');

        return () => {
            rtc.removeConnection('call');
        };
    }, [rtc, processedMicStream, volume]);

    useEffect(() => {
        void rtc.setMicStream('call', processedMicStream.current);
    }, [rtc, processedMicStream]);

    useEffect(() => {
        rtc.setVolume('call', volume);
    }, [rtc, volume]);

    return <div>Connected: {connected ? 'yes' : 'no'}</div>;
}
`}
        </RenderSource>

        ## addConnection
        `addConnection(name, options)` creates a new `WebRTCConnection`, stores it by name, and returns the created entry. If a connection with the same name already exists, the store closes the old connection before replacing it.

        ## initializeConnection
        `initializeConnection(name, offerOptions?, bearerToken?)` looks up an existing named connection in the store and calls its `initialize` method. This is the step that performs negotiation by creating the offer, waiting for ICE gathering, posting the SDP to `connectionUrl`, and applying the answer.
