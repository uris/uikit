        # WebRTCConnection Class
        Utility around `RTCPeerConnection` for one connection. It supports:
        - local audio and optional local video tracks
        - optional screen sharing
        - internal remote audio playback through an audio element
        - data channel creation and messaging
        - explicit initialization through SDP offer/answer exchange
        - replacing the outgoing microphone stream after creation
        - updating incoming playback volume independently of outgoing mic gain

        ## Import
        <RenderSource language={'tsx'}>
{`
import { WebRTCConnection } from '@apple-pie/slice/utils/objects';
`}
        </RenderSource>

        ## Basic example
        <RenderSource language={'tsx'}>
{`
const connection = new WebRTCConnection({
	connectionUrl: '/api/rtc',
	micStream: processedMicStream.current!,
	volume: 0.8,
	dataChannels: ['events'],
});

await connection.initialize(
	{
		offerToReceiveAudio: true,
		offerToReceiveVideo: true,
	},
	'my-token',
);
`}
        </RenderSource>

        ## Notes
        - `initialize()` is separate from construction on purpose. Create the object first, then negotiate when you are ready.
        - `setVolume()` only affects incoming playback. Outgoing microphone level should come from your microphone processing path.
        - `setOutgoingAudioStream()` and `replaceAudioTrack()` exist so processed mic streams can change after the connection is created.
        - The class creates outgoing data channels only. If your peer creates channels from the other side, you would need to extend the current implementation to listen for them.
        - This class is browser-only and throws when used without a document environment.

        ## Relationship To The Store And Hook
        Use `WebRTCConnection` directly when you want one low-level connection object.

        Use the WebRTC store when you want:
        - a named registry of connections
        - imperative lifecycle from React or non-React code
        - simple lookup and cleanup by connection id

        Use `useBindWebRTC` when you want:
        - explicit `bind()` / `unbind()` behavior
        - automatic microphone-store synchronization
        - automatic volume-store synchronization
        - a returned `connection` and `error` state for React integration
