        # WSConnection Class
        Typed wrapper around the browser `WebSocket` API. It supports:
        - standard message callbacks for parsed socket payloads
        - unified callback mode for `open`, `message`, `error`, and `close`
        - optional automatic reconnect on non-clean closes
        - reconnect attempt limits with optional exponential falloff
        - optional token handshake on open
        - periodic `ping` keep-alive messages

        ## Import
        <RenderSource>
```
import { WSConnection } from '@apple-pie/slice/utils/objects';
```
</RenderSource>

        ## Standard mode
        Standard mode is the default. Only message payloads flow through `onMessageCallback`. Parsed JSON is returned as `TMessage`; unparsed string or binary frames are passed through as raw data.

        <RenderSource>
```
// no typing is required, returns data as unknown
const connection1 = new WSConnection({
	url: 'wss://example.com/chat',
	onMessageCallback: (data) => {
		console.log(data);
	},
});

// optional type the message received before creating the connection
type ChatMessage = {
	id: string;
	text: string;
};

const connection2 = new WSConnection<ChatMessage>({
	url: 'wss://example.com/chat',
	onMessageCallback: (data) => {
		console.log(data.text);
	},
});
```
</RenderSource>

        ## Unified mode
        Unified mode routes `open`, `message`, `error`, and `close` through one discriminated callback. The `type` field identifies the event kind.

        <RenderSource>
```
const connection = new WSConnection<{ text: string }>({
	url: 'wss://example.com/chat',
	unifiedMessages: true,
	onMessageCallback: (message) => {
		switch (message.type) {
			case 'open':
				console.log(message.event);
				break;
			case 'message':
				console.log(message.data);
				break;
			case 'error':
				console.log(message.event);
				break;
			case 'close':
				console.log(message.event);
				break;
		}
	},
});
```
</RenderSource>

        ## Reconnect behavior
        When `autoReconnect` is enabled, `WSConnection` attempts a reconnect after any non-clean close event. Reconnects stop after `reconnectAttempts` is reached.

        If `reconnectFalloff` is enabled, the delay is exponential:
        <RenderSource>
```
delay = reconnectInterval * 2 ** (attempt - 1);
```
</RenderSource>

        Example:
        <RenderSource>
```
const connection = new WSConnection({
	url: 'wss://example.com/chat',
	autoReconnect: true,
	reconnectInterval: 1000,
	reconnectAttempts: 5,
	reconnectFalloff: true,
});
```
</RenderSource>

        Manual calls to `close()` are treated as intentional shutdown and do not trigger reconnect.

        ## Token handshake
        If a `token` is provided, it is sent as the first outbound message after the socket opens.

        <RenderSource>
```
const connection = new WSConnection({
	url: 'wss://example.com/chat',
	token: async () => {
		return await getAccessToken();
	},
});
```
</RenderSource>

        If token resolution fails, `WSConnection` emits an error event and closes the socket.

        ## Keep alive
        When `keepAlive` is enabled, the connection sends a literal `'ping'` string on the configured interval while the socket is open.

        <RenderSource>
```
const connection = new WSConnection({
	url: 'wss://example.com/chat',
	keepAlive: true,
	keepAliveInterval: 30000,
});
```
</RenderSource>

        ## Cleanup
        Always call `close()` when you are done with the connection. This clears timers, removes listeners, and closes the underlying socket.

        <RenderSource>
```
const connection = new WSConnection({
	url: 'wss://example.com/chat',
});

// later
connection.close({ code: 1000, reason: 'Component unmounted' });
```
</RenderSource>
