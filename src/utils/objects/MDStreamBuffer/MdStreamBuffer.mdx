				`MdBuffer` is a low-level utility for streamed markdown. It buffers raw chunks, emits parser-safe "healthy" markdown on animation frames, and can optimistically complete partial markdown structures while withholding ambiguous ones.
			</NoteSection>

			# MdBuffer Class
			Use `MdBuffer` when markdown streams in and you want to render structurally complete jsx/html avoiding ui "jitter".

			## Import
			<RenderSource language={'tsx'}>
            ```
            import { MdBuffer } from '@apple-pie/slice/utils/objects';
            ```
			</RenderSource>

			## What it does
			- stores the raw markdown stream
			- emits "healthy" markdown snapshots through `onFlush`
			- batches normal updates on `requestAnimationFrame`
			- commits stable line-based prefixes so only the active tail keeps being reparsed
			- supports optional link/image handling
			- supports optional HTML passthrough or best-effort stripping
			- can auto-complete and clean itself up when an end-of-stream token is received

            ## Performance note
            #### Parsing
            `MdBuffer` does not keep reparsing the full stream. When the active tail is healthy and ends on a newline boundary, the buffer promotes a stable prefix internally and only keeps parsing the remaining tail. This keeps parsing streams cheap.
            #### Rendering
            `MdBuffer` use `requestAnimationFrame` to trigger buffer flushes meaning renders will be clamped to a max of 60 per second. Flushing the buffer only happens when the buffer receives new content that has not been flushed.

			## Basic usage
			<RenderSource language={'tsx'}>
            ```
            import { useEffect, useRef, useState } from 'react';
            import { useMessage } from '@apple-pie/slice/stores';
            import { MdBuffer } from '@apple-pie/slice/utils/objects';

            function fakeParseMarkdown(markdown: string) {
                // your app would replace this with react-markdown or another renderer
                return markdown;
            }

            export function StreamingMarkdownView() {
                const message = useMessage<string>('message');
                const bufferRef = useRef<MdBuffer | null>(null);
                const [rendered, setRendered] = useState('');

                useEffect(() => {
                    bufferRef.current = new MdBuffer({
                        includeLinksAndImages: true,
                        onFlush: ({ healthy }) => {
                            setRendered(fakeParseMarkdown(healthy));
                        },
                    });

                    return () => {
                        bufferRef.current.dispose();
                        bufferRef.current = null;
                    };
                }, []);

                useEffect(() => {
                    bufferRef.current?.append(message);
                }, [message]);

                return <div>{rendered}</div>;
            }
            ```
			</RenderSource>

			## Manual flush and completion
			Use `flush()` to emit immediately without completing the stream, and `complete()` to emit a final snapshot.

			<RenderSource language={'tsx'}>
            ```
            buffer.append('This is **par');
            buffer.flush(); // emits now
            
            buffer.append('tial');
            buffer.complete(); // emits final snapshot
            ```
			</RenderSource>

			## End of stream token
			If `endOfStreamToken` is set, `append()` checks for it before normal buffering. The token is removed, the buffer completes, emits its final snapshot, and then resets itself.

			<RenderSource language={'tsx'}>
            ```
            const buffer = new MdBuffer({
                endOfStreamToken: '<eos>',
                onFlush: ({ healthy, isComplete }) => {
                    console.log(healthy, isComplete);
                },
            });
            
            buffer.append('Hello **wor<eos>');
            ```
			</RenderSource>

			## Cleanup
			Call `dispose()` when you are done with the buffer. This clears buffered state and cancels any pending animation-frame flush.

			<RenderSource language={'tsx'}>
            ```
            buffer.dispose();
            ```
			</RenderSource>
