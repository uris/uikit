        # IndexedDB Class
        Typed wrapper around the browser `indexedDB` API. It supports:
        - connecting to a named database and object store
        - optionally creating the store during initialization
        - reading one, all, or all keys
        - insert-only writes with `add`
        - upsert writes with `set` and `update`
        - deleting records, clearing a store, and deleting the database

        ## Import
        <RenderSource language={'tsx'}>
{`
import { IndexedDB } from '@apple-pie/slice/utils/objects';
`}
</RenderSource>

            <tr><td><code>clear()</code></td><td><code>{`() => Promise<void>`}</code></td><td>Deletes every record from the object store.</td></tr>
          </tbody>

        <RenderSource language={'tsx'}>
{`

await users.add({ id: 'user-2', name: 'Sam' });
await users.update('user-1', { id: 'user-1', name: 'Updated Uri' });
await users.remove('user-2');
await users.clear();
`}

        ## Notes
        - `IndexedDB` is browser-only. It throws when `indexedDB` is not available.
        - `key` is the field name used as the store key when the object store is created.
        - `add` is insert-only and rejects on duplicate keys.
        - `set` uses IndexedDB `put`, so it updates existing keys and creates missing ones.
        - `update` is currently an alias of `set`.
        - If the database exists but the store does not, creating that store requires an IndexedDB version upgrade.
