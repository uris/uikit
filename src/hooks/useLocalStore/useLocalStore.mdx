        # useLocalStore
        Persists a typed value in `localStorage` and keeps React state synchronized with the same key.
        In SSR-enabled apps, this hook also exposes when the client has hydrated so UI can avoid mismatches between the server fallback value and the browser-stored value.

        ## Signature
        `useLocalStore<T>(key, value)`

        ## Code Examples
        </Unstyled>

        <RenderSource language={"tsx"}>
        ```
        import { useLocalStore } from '@apple-pie/slice';
        
        function ProfileName() {
            const [name, setName, hydrated] = useLocalStore<string>('profile-name', 'Jane Doe');

            if (!hydrated) return null;
        
            return (
                <div>
                    <span>{name}</span>
                    <button type="button" onClick={() => setName('John Smith')}>
                        Save
                    </button>
                </div>
            );
        }
        ```
        </RenderSource>

        <Unstyled>
        ## SSR Notes
        - The server render uses the provided fallback <code>value</code>.
        - After hydration, the hook re-reads the stored browser value and flips <code>hydrated</code> to <code>true</code>.
        - Use <code>hydrated</code> when the rendered UI cannot safely show the fallback value during SSR.
        - If the value must match on the server and client first paint, use a server-readable source such as cookies instead of <code>localStorage</code>.
