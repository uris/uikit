        # useObserveTheme
        Observes `document.documentElement.dataset.sliceTheme` and returns the resolved Slice theme object.
            
        ## Signature
        `useObserveTheme()`

        ## Parameters
        This hook does not take parameters.

        ## Returns
        <table style={{width: '100%'}}>

            <col style={{width: '97%'}} />
          </colgroup>
          <thead>

              <th>Return</th>

              <th>Description</th>

            <tr><td><code>theme</code></td><td><code>SliceTheme</code></td><td><code>lightTheme</code></td><td>Current observed theme from <code>data-slice-theme</code>. On the first render, this hook uses the <code>initialTheme</code> provided by <code>ThemeProvider</code>; it only falls back to <code>lightTheme</code> when no startup theme is available.</td></tr>
          </tbody>
        </table>

        ## Code Examples
        

        <RenderSource language={"tsx"}>
        ```
        import { useObserveTheme } from '@apple-pie/slice';
        
        function ThemeBadge() {
            const theme = useObserveTheme();
            return <span>Observed theme: {theme.name}</span>;
        }
        ```
        </RenderSource>
    
        ## SSR Notes
        This hook reads the startup theme from `ThemeProvider`'s internal context before it begins observing the DOM, which keeps server-rendered and hydrated output aligned. Learn more about [Slice's ThemeServer](/docs/providers-themeserver-ssr--docs).
