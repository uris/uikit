        # useTheme
        Main Slice theming hook. Returns the active theme plus helpers and token maps for CSS variables, class names, and typography.
        In SSR-enabled apps, this hook hydrates against the `initialTheme` passed into `ThemeProvider` before switching over to live DOM observation.
        It also exposes whether the current theme is still system-derived through `systemTheme`.

        ## Signature
        `useTheme()`

        ## Parameters
        This hook does not take parameters.

        ## Code Examples
        </Unstyled>

        <RenderSource language={"tsx"}>
        ```
        import { useTheme } from '@apple-pie/slice';
        
        function ThemeToggle() {
            const theme = useTheme();
        
            return (
                <button
                    type="button"
                    className={theme.colorsClass['core-text-special']}
                    onClick={() => theme.toggle()}
                >
                    Current: {theme.current.name}
                </button>
            );
        }
        ```

        <Unstyled>
        ## SSR Notes
        - `useTheme()` does not take an `initialTheme` argument.
        - The startup value comes from `ThemeProvider`.
        - When using SSR, pass the same server-resolved `initialTheme` into `ThemeProvider` that you wrote onto the root HTML element.
        - Theme changes triggered through `set()` or `toggle()` also refresh the `slice-theme` cookie for future server renders.
        - Calling <code>set('system')</code> marks the theme as system-derived, while calling <code>set('lightMode')</code>, <code>set('darkMode')</code>, or <code>toggle()</code> marks it as explicitly selected.
