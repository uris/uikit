        # useBindWebRTC
        Binds a named `WebRTCConnection` to the microphone and volume stores with an explicit `bind()` / `unbind()` lifecycle. The hook reads the processed microphone stream from the microphone store and the current volume from the volume store, then keeps those values synchronized to one named WebRTC store connection after binding.

        This hook is intentionally a convenience layer. It does not replace the WebRTC store. If you need direct lifecycle or per-connection control, use the store actions directly.

        ## Behavior
        `useBindWebRTC` does not automatically create a connection on mount. You call `bind()` when you want the connection to be created and initialized.

        When `bind()` succeeds:
        - the hook creates a new named WebRTC store connection
        - initializes it with the provided `offerOptions` and `bearerToken`
        - keeps the outgoing audio stream in sync with the processed microphone store stream
        - keeps incoming playback volume in sync with the volume store

        When `unbind()` is called, the hook removes the named connection from the WebRTC store.

        ## Signature
        `useBindWebRTC(options)`

        ## Notes
        - `bind()` returns an error state instead of throwing when no processed microphone stream is available.
        - `bind()` also sets an error when a connection with the same `connectionName` already exists.
        - After binding, mic stream changes are pushed from the microphone store automatically.
        - After binding, volume changes are pushed from the volume store automatically.
        - `unbind()` only removes the connection when this hook created it.

        ## Basic example
        </Unstyled>

        <RenderSource language={"tsx"}>
        {`
import { useEffect } from 'react';
import { useBindWebRTC } from '@apple-pie/slice';

function CallSession() {
    const rtc = useBindWebRTC({
        connectionName: 'call',
        connectionUrl: '/api/rtc',
        offerOptions: {
            offerToReceiveAudio: true,
            offerToReceiveVideo: true,
        },
        bearerToken: 'my-token',
        dataChannels: ['events'],
    });

    useEffect(() => {
        void rtc.bind();
        return () => {
            rtc.unbind();
        };
    }, [rtc]);

    if (rtc.error) {
        return <div>Error: {rtc.error.message}</div>;
    }

    return (
        <div>
            <div>Bound: {String(rtc.bound)}</div>
            <div>Connected: {String(rtc.connection?.connected ?? false)}</div>
            <div>Audio: {String(rtc.connection?.audioConnected ?? false)}</div>
            <div>Video: {String(rtc.connection?.videoConnected ?? false)}</div>
            <div>Data: {String(rtc.connection?.dataConnected ?? false)}</div>
        </div>
    );
}
`}
        </RenderSource>

        ## When To Use The Store Directly
        Use the WebRTC store directly instead of this hook when:
        - you want to create a connection without automatically following mic store updates
        - you want to manage volume manually per connection
        - you want to attach or update connections from non-React code
        - you want to reuse an existing connection rather than requiring explicit ownership through `bind()`
